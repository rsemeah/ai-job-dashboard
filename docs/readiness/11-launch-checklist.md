# 11. Launch Checklist

## Product readiness
- [ ] Core onboarding is validated by user tests.
- [ ] End-to-end first-value flow works without manual intervention.
- [ ] Empty/loading/error/success states are complete in primary flows.

## Technical readiness
- [ ] API routes tested (unit + integration where possible).
- [ ] Role/account isolation verified.
- [ ] Observability: logs, errors, latency dashboards.
- [ ] Backup/restore process verified.

## Trust/legal readiness
- [ ] Privacy policy and terms linked and accurate.
- [ ] Data deletion/export flows available.
- [ ] Security review completed (secrets, auth, abuse controls).

## Growth/launch assets
- [ ] Landing page + messaging alignment.
- [ ] Demo narrative and walkthrough.
- [ ] App screenshots.
- [ ] Early testimonials or pilot quotes.

## GTM and support
- [ ] Support inbox + SLA defined.
- [ ] Bug triage path and severity rubric.
- [ ] Launch-week monitoring rota.
