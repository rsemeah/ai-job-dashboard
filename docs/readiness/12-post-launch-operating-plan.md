# 12. Post-Launch Operating Plan

## Cadence
- Weekly: product/analytics review
- Weekly: bug triage + SLA check
- Biweekly: user interviews and synthesis
- Monthly: roadmap reprioritization

## Core post-launch loops
1. **Signal loop:** monitor metrics + support tickets.
2. **Insight loop:** identify drop-offs and trust failures.
3. **Delivery loop:** ship fixes, then re-measure.

## Metrics to review weekly
- Activation and onboarding completion
- Time to first value
- Generation + export success rates
- Retention cohorts (D1/D7/D30)
- Churn reasons and support themes

## Incident handling
- Severity rubric (SEV1-3)
- Owner on-call rotation
- Incident timeline + postmortem template

## Experiment plan
- Onboarding simplification tests
- Document quality prompt tests
- Paywall timing and copy tests

## Missing
- Explicit numeric goals and thresholds.
- Team ownership map by metric and operational domain.
